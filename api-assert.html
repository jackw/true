    @mixin assert-true()

    Description

    Assert that a parameter is truthy. Empty lists and strings are excluded from default Sass truthyness. Assertions are used inside the test() mixin to define what will be tested.

    Parameters

    $assert: (*)

    Asserted value to test

    $description: null (string)

    Description of the assertion being tested. A null of false value generates a default description.

    requires

    @mixin _true-context() [private]

    @mixin _true-assert-results() [private]

    @function _true-is-truthy() [private]

    scss
    @include test('Non-empty strings are truthy') {
  @include assert-true('Hello World');
}

    @mixin assert()

    Description

    Define a CSS-output assertion. The assert() mixin is a wrapper, and should contain one output() block and one expect() block as nested contents. These three mixins together describe a single assert-equal comparison on output HTML. When using Mocha integration, the output comparison is automated â otherwise you will have to compare the output manually. Using git diff is a great way to watch for changes in output. The output will be compared against the results of the expect() mixin. Assertions are used inside the test() mixin to define what will be tested.

    Parameters

    $description: null (string)

    Description of the assertion being tested. A null of false value generates a default description.

    requires

    @mixin _true-output-context() [private]

    @mixin _true-context() [private]

    @mixin _true-message() [private]

    @mixin _true-update-test() [private]

    @mixin _true-update-stats-count() [private]

    @mixin _true-context-pop() [private]

    scss
    @include test('Sass math compiles before output') {
  @include assert {
    @include output {
      width: 14em + 2;
    }
    @include expect {
      width: 16em;
    }
  }
}
